# 备份、恢复与修改范围

## 修改范围

比例修改只写当前游戏 Win64 目录：

```text
winmm.dll
dlsstweaks.ini
dlsstweaks.log
```

安装时还会清理旧测试入口：

```text
dxgi.dll
nvngx.dll
XInput1_4.dll
```

如果这些旧入口存在，会先备份再删除。

## 不修改什么

本工具不会修改：

```text
NVIDIA App / DRS GlobalForcedScale
UE Engine.ini
UE ConsoleVariables.ini
启动器 LaunchCmdLine
游戏 pak 文件
```

唯一全局写入项是 DLSS HUD 开关：

```text
HKLM\SOFTWARE\NVIDIA Corporation\Global\NGXCore\ShowDlssIndicator
```

它只控制 HUD 显示，不控制 DLSS 渲染比例。

## 备份目录

每次安装都会创建：

```text
<Win64>\_nte_dlss_backups\<时间戳>
```

目录内包含：

```text
manifest.json
winmm.dll
dlsstweaks.ini
dlsstweaks.log
dxgi.dll
nvngx.dll
XInput1_4.dll
```

只有安装前实际存在的文件才会复制进去。

## manifest 作用

`manifest.json` 记录：

- 安装时间。
- 使用的比例。
- Win64 路径。
- 哪些文件安装前存在。
- 哪些文件被清理。
- 哪些文件被写入。

恢复时按这个清单操作。

## 恢复规则

如果安装前文件存在：恢复时复制回去。

如果安装前文件不存在：恢复时删除当前工具创建的同名文件。

因此恢复目标是“回到那次安装之前”，不是简单删除 `winmm.dll`。
